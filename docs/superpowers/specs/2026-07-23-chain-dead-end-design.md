# 끝말잇기 막다른 낱말 처리 — 설계

작성일: 2026-07-23
대상: `public/index.html` (끝말잇기 — 혼자하기와 같이하기 양쪽)

## 배경

끝말잇기는 정답의 끝 글자가 다음 문제가 된다. 그 글자로 시작하는 낱말이 사전에 없으면 아무도
답할 수 없다. 지금 두 모드가 다르게 반응한다.

- **같이하기**: 무한정 멈춘다. 아무도 답할 수 없는 문제가 계속 걸려 있고, 교체 요청 투표로만
  빠져나갈 수 있다. 아이들이 이 상황을 스스로 해결하기 어렵다.
- **혼자하기**: "‘X’로 시작하는 낱말을 제가 잘 몰라요. 아는 낱말을 입력해 주세요!"라고 말하고
  등록을 유도한다. 게임이 끝나지 않은 채 멈춰 있다.

같은 상황이 통합 테스트(`test/multi-race.mjs`)도 흔든다. 테스트가 이을 낱말을 찾지 못하면
예외를 던져 스위트 전체가 중단된다 — 2026-07-22 관측 기준 3회 중 1회.

## 목표

1. 막다른 낱말에 도달하면 게임을 끝내고 그 사실을 알린다. 두 모드 모두.
2. 결과를 보여준 뒤 사용자가 새 게임을 시작할 수 있게 한다.
3. 위 통합 테스트 플레이크를 없앤다.

## 비목표

- 자동 재시작하지 않는다. 결과를 볼 시간을 준다.
- 초성게임은 건드리지 않는다. 패턴은 매 라운드 새로 뽑히므로 막다를 일이 없다.
- RTDB 규칙 파일(`database.rules.json`)은 고치지 않는다.
- 혼자하기의 막다른 지점에서 낱말 등록을 유도하지 않는다. 사전에 없는 낱말을 제출했을 때의
  기존 등록 경로(`showRegisterAsk`)는 그대로 둔다 — 이번 변경과 무관하다.

## 1. 감지 — 같이하기

`hostTick`의 라운드 전환 트랜잭션에서 판단한다. 이미 `round >= MULTI_ROUNDS`면 `over`로
보내는 자리이며, 종료 조건이 하나 늘어나는 형태다.

```
reveal 3초 경과
  → 호스트의 트랜잭션
      round >= 10           → phase: 'over'      (기존, 정상 종료)
      chain이고 need가 막힘  → phase: 'over'      (신규, 막다른 종료)
      그 외                  → round + 1, phase: 'play'
```

**왜 호스트인가.** 참가자마다 사전이 다를 수 있다(`words.v1.txt` 로드 실패 시 4천 단어짜리
기본 사전만 쓰는 클라이언트가 있다 — 원장에 기록된 기존 이슈). 판단을 한 명만 하면 모두가
같은 결과를 본다. 정답 트랜잭션에서 즉시 판단하면 실행자가 정답자라서 사람마다 갈린다.

**승자 발표를 가리지 않는다.** reveal 3초는 그대로 흐르고, 그 뒤에 종료 화면으로 넘어간다.

## 2. 감지 — 혼자하기

`acceptWord`에 이미 `hasContinuation` 검사가 있다. 지금 "아는 낱말을 입력해 주세요"라고 말하는
분기에서 `endChain()`을 부른다. 종료 화면(`#chain-endbox`)과 "다시 시작" 버튼은 이미 있으므로
새 UI가 없다.

## 3. 종료 사유 구분

RTDB 규칙의 `meta`에 `"$other": { ".validate": false }`가 걸려 있어, 종료 사유를 담을 새 필드를
쓰면 **쓰기 자체가 거부된다**. 규칙을 고치는 대신 클라이언트가 추론한다.

```
phase === 'over' && round < MULTI_ROUNDS  →  막다른 종료
phase === 'over' && round >= MULTI_ROUNDS →  10라운드를 채운 정상 종료
```

라운드는 막다른 종료 시 올리지 않으므로 두 경우가 정확히 갈린다.

## 4. 문구

| | 지금 | 막다른 종료 |
|---|---|---|
| 같이하기 (`#multi-endtext`) | `게임 끝!` | `더 이을 낱말이 없어!` |
| 같이하기 순위·버튼 | 순위표 + 다시 시작 | 그대로 |
| 혼자하기 (`#chain-endtext`) | `🔗 N개 이어봤어요!` | `🔗 N개까지 이어봤어요!` |
| 혼자하기 말풍선 | `재밌었어요! 또 해요 😊` | `‘X’로 시작하는 낱말은 나도 모르겠어! 다시 할까?` |

`X`는 막힌 글자다. 혼자하기의 최고 기록 갱신 표시(`#chain-record`)는 두 경우 모두 그대로 뜬다.

`endChain`은 막힌 글자를 인자로 받아 두 경우를 가른다. 인자가 없으면 사용자가 ✕(끝내기)를
눌러 끝낸 것이므로 기존 문구를 그대로 쓴다.

```
endChain()      → 사용자가 끝냄. 기존 문구.
endChain('과')  → 막다름. 위 표의 문구.
```

같이하기의 우승자 말풍선(`OO 우승! 다시 할까?`)은 두 경우 모두 그대로 둔다. 종료 사유가
무엇이든 그 시점의 승자는 승자다.

## 5. 새 게임이 다시 막히지 않는가

시드를 뽑는 세 지점이 모두 이미 `hasContinuation`으로 거른다.

- `startChain` (혼자하기 시작)
- `changeChainSeed` (혼자하기 첫 낱말 교체)
- `newProblem` (같이하기 출제 — 최초 부트스트랩, 교체 요청 성립, `restartMulti`)

따라서 새 게임은 최소 한 수가 보장된다. `restartMulti`는 이미 `newProblem`을 거치므로 손댈 것이
없다.

## 6. 순수 함수

라운드 전환 판정을 순수 함수로 뽑아 테스트 가능하게 한다.

```
chainRoundOutcome({ mode, round, maxRounds, need, used, words }) -> 'over' | 'next'
```

```
mode가 'chain'이 아니면        → round >= maxRounds ? 'over' : 'next'
round >= maxRounds            → 'over'
need로 이을 낱말이 없으면      → 'over'
그 외                          → 'next'
```

`hasContinuation(need, words, used)`를 그대로 쓴다. `hostTick`이 이 함수의 유일한 호출자다.

## 7. 검증

**로직 테스트** — `test/logic.test.mjs`

- 초성게임은 `need`와 무관하게 라운드 수로만 끝난다.
- 끝말잇기에서 라운드가 남아 있고 이을 낱말이 있으면 `'next'`.
- 끝말잇기에서 라운드가 남아 있어도 이을 낱말이 없으면 `'over'`.
- 이미 쓴 낱말(`used`)밖에 없으면 이을 수 없는 것으로 본다.
- 마지막 라운드는 이을 낱말이 있어도 `'over'`.

**통합** — `test/multi-race.mjs`

기존 플레이크의 원인을 제거하는 방향으로 바꾼다. 지금은 이을 낱말을 못 찾으면 예외를 던져
스위트가 중단되는데, 이제 그것은 정상 동작이므로 예외 대신 종료 화면을 확인한다.

1. 막다른 낱말을 만나면 양쪽 브라우저가 `over`로 넘어간다.
2. 종료 화면에 `더 이을 낱말이 없어!`가 뜬다.
3. "다시 시작"을 누르면 새 게임이 시작되고, 새 문제는 이을 낱말이 있다.

**회귀**: 기존 로직 테스트 79개 전량 통과, 초성게임 시나리오 무손상, 혼자하기의 사전에 없는
낱말 등록 경로 무손상.
