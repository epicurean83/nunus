# 같이하기 자동 게임 종료 제거 — 설계

작성일: 2026-07-23
대상: `public/index.html` (끝말잇기 같이하기)

## 배경

2026-07-23에 끝말잇기 막다른 낱말 처리를 넣었다(`feature/chain-dead-end`). 같이하기에서는
호스트가 라운드를 넘길 때 자기 사전으로 "이 글자로 이을 낱말이 있나"를 판단해, 없으면 게임을
끝냈다.

그 판단이 형평성에 어긋난다. 최종 리뷰가 실측한 결과, 37k 사전에서 이을 수 있는 끝 글자 789개
중 238개(30%)가 4k 내장 사전에서는 막다르게 보인다. `words.v1.txt` 로드에 실패한 클라이언트가
호스트가 되면, 다른 참가자는 사전에 그 낱말을 갖고 있어 이어갈 수 있는데도 방 전체가 조기
종료된다. 한 사람의 사전이 모두의 게임을 끝내는 셈이다.

## 목표

같이하기에서 자동 게임 종료를 제거한다. 막다른 상황은 기존 '문제 교체 요청' 투표로 해결한다 —
참가자가 함께 정하므로 한 사람의 사전이 결정하지 않는다.

## 비목표

- 혼자하기의 막다른 종료는 그대로 둔다. 혼자는 한 명이라 형평성 문제가 없고, 사용자가 멀티에
  한정해 제거를 요청했다.
- 교체 투표 자체는 건드리지 않는다. 이미 막다름의 탈출구다.
- 새 기능을 넣지 않는다. 자동 교체 같은 대체 동작은 추가하지 않는다(형평성 문제가 옅게 남고,
  사용자가 교체 투표에 맡기기로 정했다).

## 되돌릴 것

`feature/chain-dead-end`가 같이하기에 넣은 것만 걷어낸다.

1. **`chainRoundOutcome` 제거.** 같이하기의 `hostTick`이 유일한 호출자였다. 혼자하기는 이 함수를
   쓰지 않고 `acceptWord`에서 `hasContinuation`을 직접 부른다. 호출자가 사라지면 죽은 코드이므로
   CORE에서 함수를, `test/logic.test.mjs`에서 `CORE_NAMES` 등록과 테스트 6개를 지운다.
   (로직 테스트 85 → 79)

2. **`hostTick` 라운드 전환을 옛 판정으로 되돌린다.** `chainRoundOutcome` 호출과 `outcome === 'over'`
   분기를 지우고, 원래의 라운드 수 판정만 남긴다.

   ```js
       if(cur.round >= MULTI_ROUNDS){ cur.phase = 'over'; return cur; }
       cur.round = cur.round + 1;
   ```

3. **`renderMulti`의 종료 문구를 되돌린다.** `deadEnd` 추론을 지우고 `'게임 끝!'`로 되돌린다.

## 남길 것

되돌리지 않는다 — 이번 작업과 무관하거나 독립적으로 옳은 것들이다.

- **`hostTick`의 `capturedMode`.** 트랜잭션 콜백이 재실행되며 호스트가 나가면 `MG`가 null이 되어
  `MG.mode` 접근이 터지던 것을 막은 수정이다(막다른 종료와 무관한 기존 버그 수정). `chosung`
  분기가 여전히 이 값을 쓴다. 그대로 둔다.
- **혼자하기 `endChain(deadEnd)`와 `acceptWord`의 막다른 분기.** 혼자하기 종료는 유지한다.
- **통합 테스트의 플레이크 제거(레이스 루프).** 이을 낱말을 못 찾을 때 예외로 스위트를 죽이던
  것을, 방을 끝내고 재시작 경로로 보내도록 바꾼 부분은 그대로 둔다. 자동 종료가 사라져도 레이스
  워크가 막다른 need에 도달할 수 있고, 그때 테스트가 스스로 진행을 만들어야 하기 때문이다.

## 검증

**로직 테스트** — `chainRoundOutcome` 테스트 6개를 지운 뒤 나머지 79개가 전부 통과한다.

**통합** — `test/multi-race.mjs`

- **3.8 시나리오 교체.** 지금은 "막다르면 게임이 끝난다"를 검증한다. 이제는 그 반대를 고정한다:
  이을 낱말이 없는 need를 심고 라운드 전환을 태워도 게임이 `over`로 가지 않고 `play`로 진행되며,
  교체 투표 버튼이 여전히 있다.
- fail-before(되돌리기 전, 자동 종료가 있는 상태): 게임이 `over`로 가서 "play로 진행" 단언이
  실패한다. pass-after: 통과한다.

**회귀** — 혼자하기 막다른 종료 무손상, 초성게임 무영향.

## 변경 범위

- `public/index.html` — `chainRoundOutcome` 삭제, `hostTick` 되돌리기, `renderMulti` 문구 되돌리기.
- `test/logic.test.mjs` — `CORE_NAMES` 등록과 테스트 6개 삭제.
- `test/multi-race.mjs` — 3.8 시나리오를 "자동 종료 안 함" 검증으로 교체.
